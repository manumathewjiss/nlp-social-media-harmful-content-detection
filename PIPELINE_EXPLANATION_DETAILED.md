# Detailed Pipeline Explanation in Words

This document provides a very long, detailed explanation of the entire pipeline of the project in continuous prose. It walks through every phase, every decision, and every technical step so that the reader can understand exactly what was done from start to finish.

---

## 1. Where the Pipeline Begins: The YouTube Comments Dataset

The pipeline starts with a large collection of YouTube comments. This dataset is stored in a CSV file and contains two main columns: the raw comment text and a sentiment label. The sentiment label is one of three classes: positive, neutral, or negative. This dataset is the backbone of the first part of the project. Importantly, the dataset is not balanced. In the real world, comment sections tend to have more positive or neutral comments than negative ones, and this dataset reflects that reality. Specifically, the original dataset contains approximately 18,408 comments. Of these, the vast majority are positive (around 11,432 comments, or about 62 percent). The next largest group is neutral (about 4,638 comments, or 25 percent), and the smallest group is negative (about 2,338 comments, or 13 percent). This imbalance is deliberate: we want to see how well our models perform on data that looks like real-world comment sections, where one class dominates. Before any model sees this data, every comment goes through a shared preprocessing stage, which is described next.

---

## 2. Text Preprocessing: Cleaning the Comments for All Phases

Before any comment is fed into any model anywhere in the pipeline, it is cleaned and normalized. This preprocessing is the same for YouTube comments (both imbalanced and balanced), for the negative comments that go to the toxicity classifier, for the Instagram post text, and for any other text input. The goal is to remove noise that might confuse the models and to keep the input format consistent.

First, each piece of text is converted to a string and then converted to lowercase. This reduces the vocabulary size and ensures that the same word in different cases is treated the same way. Next, all URLs are removed. Any substring that looks like a web link (starting with http, https, or www) is stripped out, so that the models focus on the actual language of the comment and not on links. After that, user mentions and hashtags are removed. In social media text, @username and #hashtag are common; these are removed using regular expressions so that they do not influence sentiment or toxicity labels. Then, special-character normalization is applied: any character that is not a letter (a–z, A–Z), a digit (0–9), or one of a small set of punctuation marks (space, comma, period, exclamation, question mark, apostrophe) is replaced by a single space. This gets rid of emojis, unusual symbols, and other non-standard characters. Finally, consecutive spaces are collapsed into one space, and leading and trailing whitespace is stripped. If after all these steps a comment becomes empty, it is dropped from the dataset. The result is a clean text field (often called clean_text) that is used for all downstream sentiment and toxicity models. This ensures that every phase of the pipeline sees the same kind of input.

## 3. Phase 1: Sentiment Analysis on YouTube and the Creation of a Balanced Dataset

The first major phase of the project is three-class sentiment classification on YouTube comments. We want to predict whether each comment is positive, neutral, or negative. To do this, we use pre-trained transformer models that were already trained for this task on large Twitter or tweet corpora. We do not train these models ourselves; we only run them in inference mode. Two such models are used: BERTweet and RoBERTa.

BERTweet is a BERT-style model (finiteautomata/bertweet-base-sentiment-analysis) that was pre-trained on tweets and then fine-tuned for three-way sentiment. RoBERTa (cardiffnlp/twitter-roberta-base-sentiment) is a Robustly Optimized BERT pretrained on a very large corpus of tweets (on the order of 124 million tweets from 2018 to 2021) and fine-tuned for the same three-way sentiment task. Both models take a sequence of tokens as input and output a distribution over the three labels; we take the label with the highest score as the prediction. For both models, we use the same inference settings: maximum sequence length of 128 tokens, batch size of 16, and padding and truncation applied to each batch. The tokenizer and model are loaded (with an optional local cache directory to avoid re-downloading), and we run through the dataset in batches without any further training.

We do not only run these models on the imbalanced dataset. We also create a second dataset, called the balanced dataset, and run the same models on it. The reason is to compare how the models behave when the classes are balanced versus when they are imbalanced. The balanced dataset is created from the original YouTube comments as follows. We set a target number of samples per class (for example, 4,500 per class). For each sentiment class, we look at how many samples we already have. If we have fewer than the target, we generate new synthetic samples until we reach the target. To generate a synthetic sample, we pick a random comment from that class and apply one of four augmentation methods. The first method is synonym replacement: we have a predefined dictionary that maps sentiment-bearing words (such as “bad”, “good”, “hate”, “love”, “amazing”, “terrible”, “great”) to lists of synonyms, and we randomly replace up to two words in the comment with a synonym from the list, preserving punctuation. The second method is phrase addition: we prepend or append short phrases to the comment depending on the sentiment (for example, “Wow! ” or “ Love it!” for positive, “Ugh! ” or “ Hate it!” for negative, “Well, ” or “ Not sure.” for neutral). The third method is paraphrase: we apply simple regex-based substitutions (for example, “I think” to “I believe”, “very” to “really”) once per comment. The fourth method is a combination: we first apply synonym replacement and then phrase addition. We cycle through these methods so that we generate a mix of augmentation types. Once we have enough samples per class (4,500 each in our case), we combine all samples, shuffle the dataset with a fixed random seed (42) for reproducibility, and save it as a new CSV file (YoutubeCommentsDataSet_Balanced.csv). The final balanced dataset has 13,500 comments in total (4,500 negative, 4,500 neutral, 4,500 positive).

We then run both BERTweet and RoBERTa on both datasets. That gives us four experimental conditions: BERTweet on the imbalanced dataset, BERTweet on the balanced dataset, RoBERTa on the imbalanced dataset, and RoBERTa on the balanced dataset. For each condition, we compute overall accuracy, per-class precision, recall, and F1-score, macro-averaged metrics, and the 3×3 confusion matrix (rows = actual sentiment, columns = predicted sentiment). A comparison script loads the saved prediction files for all four conditions and produces a numerical report and visualizations (for example, a bar chart of accuracy across the four conditions and separate confusion matrices for each). After this comparison, we select the single best condition for the rest of the pipeline. In our experiments, RoBERTa on the imbalanced dataset achieved the highest accuracy (90.23 percent on 18,214 evaluated samples). The per-class performance was as follows: for the negative class, precision was 0.8966, recall 0.8900, and F1-score 0.8933 (support 2,337); for the neutral class, precision 0.7589, recall 0.9037, F1-score 0.8250 (support 4,463); for the positive class, precision 0.9755, recall 0.9042, F1-score 0.9385 (support 11,414). We therefore select RoBERTa on the imbalanced dataset as our single sentiment model for all downstream steps that use YouTube sentiment.

From this selected run, we extract every comment that was predicted as negative. We save these comments together with their preprocessed text (clean_text), original comment, and any identifiers to a dedicated CSV file (phase2_input_negative_comments_roberta_imbalanced.csv). The number of negative comments extracted is 2,320, which is about 12.7 percent of the evaluated dataset. This file becomes the sole input to the next phase. We do not pass the balanced dataset or any synthetic comments to the next phase; only the negatives from the RoBERTa imbalanced run are used.

## 4. Phase 2: Zero-Shot Toxicity Classification with BART

The second phase takes the 2,320 negative comments from Phase 1 and assigns each one to exactly one of seven toxicity-related categories. We do not train a new model for this. Instead, we use a zero-shot classification model: BART Large MNLI (facebook/bart-large-mnli). This model was trained for natural language inference and can be used for zero-shot classification by treating the input text and each candidate label as a premise-hypothesis pair and scoring how much the premise entails the hypothesis. The Hugging Face pipeline API wraps this so that we can pass a comment and a list of candidate labels and get back a score for each label; we take the label with the highest score as the prediction.

The seven categories we use are: harassment or hate speech, spam, inappropriate content, toxicity, aggressive behavior, misinformation, and other negative. These are passed to the pipeline as a single list, and we set multi_label to false so that each comment gets exactly one predicted category. We process each row of the Phase 2 input CSV one by one: we take the preprocessed text (clean_text), pass it to the zero-shot pipeline with these seven labels, and record the top-scoring label and its confidence score. We then map the internal label names to readable names (for example, “Harassment/Hate Speech”, “Spam”, “Inappropriate Content”, “Toxicity”, “Aggressive Behavior”, “Misinformation”, “Other Negative”) and append the predictions and confidence scores to the dataframe. We do not have ground-truth toxicity labels for this phase; the output is a distribution of predicted categories and summary statistics of confidence. The results show that the majority of comments (1,358, or 58.53 percent) were classified as “Other Negative”, meaning they did not fit neatly into the six specific toxicity types. The next largest categories were Aggressive Behavior (335 comments, 14.44 percent), Misinformation (279, 12.03 percent), Inappropriate Content (223, 9.61 percent), Toxicity (95, 4.09 percent), Harassment/Hate Speech (24, 1.03 percent), and Spam (6, 0.26 percent). The average confidence was 50.78 percent, the median 48.80 percent, with a standard deviation of 16.39 percent and a range from 18.72 percent to 97.33 percent. We save the classification results, generate a category distribution chart, a classification matrix visualization, and confidence distribution plots. This phase is used for analysis and reporting only; its output is not fed into the Instagram or LIMFAAD parts of the pipeline.

## 5. The YouTube Synthetic Balanced Dataset: A Separate Evaluation

In parallel to the main pipeline that goes from Phase 1 to Phase 2, we perform a separate evaluation using the synthetic balanced YouTube dataset. The goal is to see how RoBERTa performs when the classes are balanced (4,500 per class) and to compare that performance to the performance on the original imbalanced dataset. We do not send the balanced dataset through BART; this is only about sentiment.

The balanced dataset is the same one created for Phase 1: 13,500 comments (4,500 per class) built by augmenting the original data with synonym replacement, phrase addition, paraphrase, and combo methods. We load this CSV, apply the same preprocessing to each comment, and run the same RoBERTa model (cardiffnlp/twitter-roberta-base-sentiment) with the same settings (batch size 16, max length 128). We compare the predictions to the ground-truth Sentiment column in the balanced CSV and compute accuracy, per-class precision/recall/F1, macro and weighted metrics, and the confusion matrix. We also optionally load the metrics from the original (imbalanced) RoBERTa run and print a side-by-side comparison (original versus synthetic) for accuracy and macro F1. This tells us whether balancing the dataset improves or changes RoBERTa’s behavior. The confusion matrix and metrics visualizations (for example, bar charts for precision, recall, and F1 by class) are saved (e.g. synthetic_dataset_confusion_matrix.png, synthetic_dataset_metrics_visualization.png). This whole step is standalone: it does not feed into BART or into the Instagram or LIMFAAD pipeline. It is purely an evaluation of RoBERTa on synthetic balanced data.

## 6. The Instagram Synthetic Dataset and RoBERTa Sentiment Analysis

The project then moves to a different platform and dataset: Instagram. We have a synthetic dataset of Instagram posts or comments (for example, 150 entries) stored in a CSV file (e.g. task1_dataset/InstagramPosts_Base.csv). Each row has at least a text column (e.g. Post_Text) and user-profile fields such as followers, following, posts, bio, profile picture, external link, threads, and mutual friends. The goal here is to run sentiment analysis on the text only (we do not use user details for sentiment), and then to keep only the comments that are predicted as negative, along with their full user-profile metadata, for a later step where we will classify the accounts.

We load the Instagram CSV and extract the text column (Post_Text). Missing values are filled with an empty string, and then we apply the same shared preprocessing function (lowercasing, URL removal, mention/hashtag removal, special-character normalization, whitespace cleanup). Rows that end up with empty cleaned text are dropped. We then load the same RoBERTa model and tokenizer used in Phase 1 (cardiffnlp/twitter-roberta-base-sentiment), with the same batch size (16) and max length (128), and run inference on all rows. Each row gets a predicted sentiment: negative, neutral, or positive. We save the full results (all rows with their predictions) and also create a separate file that contains only the rows predicted as negative, preserving all original columns including the user-profile fields. In our run, there were 150 posts/comments; 59 were predicted positive (39.3 percent), 55 negative (36.7 percent), and 36 neutral (24.0 percent). The 55 negative comments, together with their user details (followers, following, posts, bio, profile picture, external link, threads, mutual friends), are saved to a file such as task2_roberta/instagram_roberta_negative_comments.csv. We do not run BART or any other toxicity classifier on the Instagram data. This negative-comments file with user details is the input to the final classification phase.

## 7. The LIMFAAD Dataset and Training the XGBoost Account-Type Classifier

The next phase trains a classifier to predict account type from profile-derived features only (no comment text). We use the LIMFAAD dataset (stored in a file such as LIMFADD.csv), which contains many thousands of accounts (in our case 15,000 samples) with profile features and a label indicating account type. The labels are four classes: Bot, Scam, Real, and Spam. The feature columns include: Followers, Following, Following/Followers (a ratio), Posts, Posts/Followers (a ratio), Bio (binary or categorical), Profile Picture (binary), External Link (binary), Mutual Friends, and Threads (binary). Some cells may contain invalid values such as “#DIV/0!” when denominators are zero; we fix these by replacing them appropriately (e.g. when Followers is zero, Following/Followers is set to a large value or the maximum of Following, and Posts/Followers is set to zero), and we fill any remaining missing values with the column median. We encode categorical columns to binary (e.g. “yes” or “y” to 1, else 0) for Bio, Profile Picture, External Link, and Threads. The label column is encoded with a scikit-learn LabelEncoder for training.

We split the dataset into training and test sets (80 percent train, 20 percent test) with stratification on the label and a fixed random state (42). That gives 12,000 training samples and 3,000 test samples (750 per class). We then train an XGBoost classifier for multi-class classification. The main parameters we use are: objective multi:softprob, num_class 4, eval_metric mlogloss, max_depth 6, learning_rate 0.1, n_estimators 200, subsample 0.8, colsample_bytree 0.8, random_state 42. We fit the model on the training set; optionally we hold out a small validation set from the training set for monitoring. After training, we evaluate on the test set. The model achieved a training accuracy of 99.69 percent and a test accuracy of 96.90 percent. Per-class performance on the test set (750 samples per class) was: Bot — precision 0.9960, recall 0.9853, F1-score 0.9906; Scam — precision 0.9855, recall 0.9973, F1-score 0.9914; Spam — precision 0.9432, recall 0.9520, F1-score 0.9476; Real — precision 0.9515, recall 0.9413, F1-score 0.9464. Macro-averaged precision, recall, and F1-score were all 0.9690. We also compute feature importance: the most important features were Followers (importance about 0.38), Posts (about 0.30), Following (about 0.10), Mutual Friends (about 0.07), and the ratio features; binary features like Bio, External Link, Threads, and Profile Picture had lower importance. We save the trained model (e.g. limfaad_xgboost_model.json), the label encoder (limfaad_label_encoder.pkl), and a small artifact with the list of feature column names and class names (limfaad_feature_info.pkl) so that we can load and use the model later with exactly the same feature order and label mapping.

## 8. Classifying the Instagram Negative-Comment Accounts with the Trained XGBoost Model

The final phase applies the LIMFAAD-trained XGBoost model to the 55 Instagram accounts that posted negative comments (from the file produced in the Instagram RoBERTa step). We do not use the comment text for this step; we use only the user-profile fields, and we must represent them in exactly the same way as in the LIMFAAD training data.

We load the negative-comments CSV (e.g. task2_roberta/instagram_roberta_negative_comments.csv), which has one row per account (or per comment–account pair) with columns for followers, following, posts, bio, profile picture, external link, threads, mutual friends, and possibly others. We parse numeric fields from strings (e.g. “1.2k” to 1200, “1M” to 1,000,000). If a Posts column is missing, we set it to zero. We compute the same ratios as in LIMFAAD: Following/Followers and Posts/Followers, using the same logic for zero followers (NaN then fill with median or constant). We encode binary/categorical columns (Bio, Profile Picture, External Link, Threads) to 0/1 with the same rules as in training. Mutual Friends, if present, is parsed to numeric; if missing, we set it to 0. The result is a feature matrix with exactly the same columns in the same order as stored in limfaad_feature_info.pkl.

We load the trained XGBoost model and the label encoder from the task3_limfaad models directory, pass the aligned feature matrix through the model, and obtain predicted class labels (and optionally per-class probabilities). We decode the numeric predictions back to the original class names (Bot, Scam, Real, Spam) using the saved encoder. We write the results to a CSV that includes the original columns plus predicted label and confidence (maximum probability). In our run, all 55 accounts were classified: 30 as Real (54.5 percent), 15 as Spam (27.3 percent), 6 as Bot (10.9 percent), and 4 as Scam (7.3 percent). So more than half of the accounts that posted negative comments were classified as authentic (Real), while about 45.5 percent were classified as potentially problematic (Spam, Bot, or Scam). We also generate a confusion matrix plot (if ground-truth labels are available), a category distribution plot, and metrics visualizations, and save them under task4_classification. This completes the pipeline: we have sentiment from RoBERTa and account-type from XGBoost for the Instagram synthetic accounts that posted negative comments.

## 9. Summary: How the Pieces Fit Together

In words, the full pipeline is as follows. We start with a large imbalanced YouTube comments dataset (about 18,408 comments: mostly positive, then neutral, then negative). Every comment is preprocessed the same way (lowercase, no URLs/mentions/hashtags, normalized characters, cleaned spaces). We also build a balanced version (13,500 comments, 4,500 per class) using augmentation (synonym, phrase, paraphrase, combo). We run BERTweet and RoBERTa on both the imbalanced and balanced datasets, compare the four conditions, and choose RoBERTa on the imbalanced dataset (90.23 percent accuracy). We extract the comments RoBERTa predicted as negative (2,320) and send them to a BART zero-shot model, which assigns each to one of seven toxicity categories (most fall into “Other Negative”). Separately, we run RoBERTa on the balanced YouTube dataset and compare metrics to the imbalanced run. In parallel, we take an Instagram synthetic dataset (150 posts), run RoBERTa on the post text only, and keep the 55 comments predicted as negative along with their user-profile metadata. We train an XGBoost classifier on the LIMFAAD dataset (15,000 samples, 10 profile features, 4 classes: Bot, Scam, Real, Spam) and achieve 96.90 percent test accuracy. Finally, we take the 55 Instagram accounts with negative comments, align their profile fields to the LIMFAAD feature set, run the trained XGBoost model, and obtain account-type predictions: 54.5 percent Real, 27.3 percent Spam, 10.9 percent Bot, 7.3 percent Scam. From start to finish, the pipeline moves from raw YouTube and Instagram text through preprocessing, sentiment (RoBERTa), toxicity (BART on YouTube negatives only), synthetic evaluation (RoBERTa on balanced YouTube), and account-type classification (XGBoost on LIMFAAD and then on Instagram negative-comment accounts), with every step described in detail above.
